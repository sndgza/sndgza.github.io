shell学习（1）变量

发表于 2021-11-15 更新于 2021-11-22 分类于 linux-shell 阅读次数： Valine：
本文字数： 1.1k 阅读时长 ≈ 1 分钟

shell 脚本的定义与执行

1、定义开头： #!/bin/bash
#！用来声明当前脚本文件的shell解释器的类型，如果不写则为默认的shell
2、编写代码 单个#表示注释
3、执行
chmod +x xxx.sh ./xxx.sh 修改为可执行权限后执行 检测#!，使用#！指定的bash读取解释，没有则使用默认的bash
bash test.sh 直接指定使用bash解释xxx.sh
. test 使用当前shell读取接收xxx.sh

定义变量
变量名=变量值
#赋值操作时不能在=两边加空格
num=10
引用变量
$变量名
如：i=$num
显示变量
echo $num
echo $i
清除变量
unset num

变量的其他用法
read string
从键盘输入字符串给string
readonly var=100
只读变量，以后不能改变不能清除
export var=300
使用export说明的变量，会被导出为环境变量，其他shell均可使用

shell环境变量

shell在开始执行时就已经定义了一些和系统的工作环境有关的变量，我们在shell中可以直接使用$name引用
定义：
一般在~/.bashrc或/etc/profile文件中（系统自动调用的脚本）使用export设置，允许用户后来更改
VARNAME=value;export VARNAME
传统上，所有环境变量均为大写
显示环境变量
使用env可以查看当前系统的所有环境变量
清除环境变量
unset清除环境变量
常见环境变量：

HONE：保存注册目录的完全路径名
PATH：保存用：分隔的路径名，shell将按PATH变量中给出的顺序搜索这些目录，找到的第一个与名称一致的可执行文件将被执行
HOSTNAME：主机名
SHELL：默认的shell命令解释器
LOGAME：此变量保存登陆名
PWD：当前工作目录的绝对路径名
….
在终端定义的环境变量只是临时的，在配置文件~/.bashrc中添加的环境变量才能保存不变（sourc ~/.bashrc）

预设变量

$ ：（超过10需要加{}例如：echo”$10 = ${10}”）
$0：当前的进程名
$#：参数个数
$$：当前进程号
$*：传给shell脚本参数的内容
$?: 检查上一条命令执行是否正确，如果成功返回0，不成功返回非0。

脚本变量的特殊用法

“”：包含的变量会被解释
‘’：包含的变量会被当做字符串解释
``：反引号中的内容作为系统命令，并执行其内容，可以替换输出为一个变量（等价于$()）
(命令序列)：由子序列完成，不影响当前shell的值
{命令序列}：当前shell执行，影响当前shell的值
-e：使用转义字符
